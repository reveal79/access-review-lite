# Contributing

Issues and focused pull requests are welcome.

1. Keep Graph access read-only.
2. Add or update Pester tests for behavioral changes.
3. Run PSScriptAnalyzer with Error and Warning severities.
4. Run the complete Pester suite.
5. Generate the demo report and inspect it before submitting UI changes.

Permission-catalog changes must include a clear rationale and remediation statement. Missing or inaccessible data must never be converted into a risk finding.
